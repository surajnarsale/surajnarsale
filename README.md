### 🧠 How I Work

I write code like I architect — modular, testable, and hard to break.  
I enjoy solving the problems most people skip over:  
performance bottlenecks, tangled state, poor DX, brittle component boundaries, and architectural sprawl across teams.

I care about:

- clean separation between UI, state, and data layers  
- scaling frontends with design systems, microfrontends, and clear contracts  
- CI/CD pipelines and containerized environments that stay predictable  
- writing tests that catch real bugs, not just pad coverage numbers  
- using AI tools (Claude Code, Copilot, Cursor) to ship faster without cutting corners  

Mostly, I build in **React**, **Next.js**, **Vue**, **Angular**, and **TypeScript**.  
Backend when needed — **Node.js**, **Python**, **.NET**.  
Always keeping **Docker**, **Kubernetes**, and **CI/CD** close by.
